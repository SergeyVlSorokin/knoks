# 04 — Manage Clients and derived first-use state

**What to build:** An Administrator can operate the complete Client lifecycle through ordinary product views, while Administration and My time always derive accurate setup guidance from retained Client records rather than a separate setup flag.

**Blocked by:** 01 — Provision and enter the Company Workspace.

**Status:** ready-for-agent

- [ ] With no retained Clients, Administration gives equal prominence to Accounts and Clients and explains that an active Client is required before time can be recorded.
- [ ] An Administrator can create a Client with a required trimmed display name; uniqueness is case-insensitive across active and archived Clients while chosen capitalization is preserved.
- [ ] Creating the first active Client immediately removes first-use guidance without a separate completion action.
- [ ] An Administrator can rename a Client without changing its identity, and current views show the current name.
- [ ] Archiving removes a Client from new attribution while preserving history and Invoice Basis eligibility; restoration makes it available again.
- [ ] Permanent deletion succeeds only before any Time Entry has ever referenced the Client.
- [ ] Retained but all-archived Clients produce operational Create and Restore actions rather than first-use wording.
- [ ] A Member with no active Clients sees an actionable My time empty state and cannot open an entry grid.
