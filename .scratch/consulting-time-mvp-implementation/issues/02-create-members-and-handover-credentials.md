# 02 — Create Members and hand over credentials

**What to build:** An Administrator can create a permanent Member or Administrator account, securely hand over a generated initial password shown once, and leave the new person able to sign in and maintain their own password.

**Blocked by:** 01 — Provision and enter the Company Workspace.

**Status:** ready-for-agent

- [ ] An Administrator can create an account with a trimmed display name, workspace-unique username, and Member or Administrator role.
- [ ] Account creation generates an initial password and displays it exactly once in a credential receipt.
- [ ] Dismissing the credential receipt leaves the account created and never reveals the password again.
- [ ] The new account can sign in with the generated password and can change its own password.
- [ ] Member access is limited to active Clients needed for entry, the Member's profile, and that Member's own Time Entries and history.
- [ ] An Administrator can access all account administration while retaining Member abilities.
